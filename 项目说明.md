# 宠物社交平台（PetSocial）项目说明

## 一、项目背景与意义

随着宠物经济的持续升温，"铲屎官"群体对宠物日常分享、经验交流、线下活动以及宠物相关消费的需求日益增长，但目前主流社交平台缺乏面向宠物垂直场景的深度功能。基于此，本项目设计并实现了一个面向宠物主人的垂直社交平台 **PetSocial**，提供"宠物身份社交 + 内容分发 + 商家服务 + 积分激励 + 线下活动"的一体化解决方案，旨在打通"线上分享—社区互动—线下消费"的完整链路。

## 二、项目目标

1. 以"宠物"为社交主体，构建有别于传统人际社交的内容生态。
2. 提供跨平台客户端（H5 / 微信小程序 / App）以满足不同使用场景。
3. 支持商家入驻、礼品兑换与线下活动报名，形成业务闭环。
4. 提供完善的后台管理能力以保证内容质量与平台安全。
5. 集成宠物百科微服务，提升平台的内容深度与专业性。

## 三、系统架构

项目采用 **多端分离 + 微服务** 架构，由四个独立模块组成：

| 模块 | 技术栈 | 端口 | 职责 |
|------|--------|------|------|
| `server/` | Express 5 + MySQL + Sequelize | 3001 | 核心业务 REST API |
| `frontend/` | UniApp + Vue 3 | 动态 | 跨平台移动客户端（H5 / 微信小程序 / App） |
| `admin/` | Vue 3 + Vite + Element Plus | 5173 | Web 后台管理系统 |
| `dataDB/` | Express + MongoDB + Mongoose | 3005 | 宠物百科数据微服务 |

**架构亮点：**

- **关系型 + 文档型**双数据库：社交关系数据使用 MySQL，宠物百科类非结构化文档使用 MongoDB，做到合适的数据用合适的库。
- **前后端完全解耦**：通过 RESTful API 通信，admin 端通过 Vite 代理转发请求避免开发期 CORS 问题。
- **一套代码多端运行**：UniApp 框架使前端单代码库即可编译至 H5、微信小程序、原生 App 等多平台。

## 四、核心功能模块

### 1. 账号与宠物身份系统

- 邮箱注册、JWT 登录鉴权（有效期 7 天）。
- 通过 Nodemailer 发送邮箱验证码，支持注册校验与密码重置。
- `Account` 与 `Pet` 一对一关联，用户以"宠物"身份在平台中活动。

### 2. 内容发布与互动

- 帖子的 CRUD、图片上传（基于 Multer）。
- 多维度反馈：点赞（PostLike）、踩（PostDislike）、收藏（Bookmark）、不感兴趣过滤、评论嵌套。
- 屏蔽机制（BlockPet）：拉黑后双向不可见，提升用户体验。

### 3. 关注与社交关系

- `Follow` 表为 Pet ↔ Pet 的自连接，支持双向关注列表查询。

### 4. 话题与分类

- 帖子与话题为 **多对多** 关系（`PostTopic` 中间表），支持一帖多标签。
- 系统启动时自动初始化"未分类"等 7 类基础分类，以及 5 个默认话题。

### 5. 积分体系

- 用户通过发帖、互动等行为获得积分，所有积分变动写入 `PointLog` 形成可追溯流水。
- 积分可在礼品商城进行兑换，已修复"兑换后积分未实时更新"的体验问题。

### 6. 商家与礼品商城

- `Merchant`：商家入驻申请、状态审核。
- `Gift`：商家发布的可兑换礼品。
- `GiftRedemption`：用户兑换记录，支持商家端订单管理。

### 7. 线下活动

- `Activity`：商家发布线下活动。
- `ActivityParticipant`：用户报名记录，支持活动详情、报名与列表浏览。

### 8. 宠物百科（独立微服务）

- 由 `dataDB/` 提供，基于 MongoDB 存储非结构化的百科条目，前端通过独立接口访问。

### 9. 后台管理系统

- 基于 Element Plus 实现，包含 Login / Layout / PostList / CategoryList / TopicList / CommentList / MerchantList / ActivityList 等视图，覆盖内容审核、商家审核、活动管理等管理需求。

## 五、关键数据模型

数据库共计 **19 个表**，核心实体关系：

- `Account` ↔ `Pet`：1:1（账户—宠物身份）
- `Pet` → `Post`：1:N
- `Post` ↔ `Topic`：M:N（通过 `PostTopic`）
- `Pet` ↔ `Pet`：M:N 关注 / 拉黑
- `Account` → `PointLog`：1:N（积分流水）
- `Merchant` → `Gift` → `GiftRedemption`：商家—礼品—兑换链路
- `Merchant` → `Activity` → `ActivityParticipant`：商家—活动—参与链路

模型关系统一定义于 `server/src/models/index.js`，启动时通过 `sequelize.sync({ alter: true })` 自动建表并初始化基础数据与管理员账号。

## 六、关键技术与实现要点

1. **JWT 鉴权中间件**：`server/src/middleware/auth.js` 统一拦截 `Authorization: Bearer <token>` 头，过期返回 401 触发前端登出。
2. **Sequelize ORM**：使用关联（`belongsTo` / `hasMany` / `belongsToMany`）替代手写 SQL，提升可维护性。
3. **请求统一封装**：`frontend/src/utils/request.js` 包装 `uni.request`，自动注入 Token、统一处理 `code=0` 成功 / `code=401` 登出。
4. **图片资源服务**：通过 `express.static` 暴露 `src/public/images/` 目录提供上传文件访问。
5. **数据自举**：服务启动时自动创建数据库结构、初始化分类/话题/管理员账号，开箱即用。
6. **跨平台编译**：UniApp 一套代码同时支持 H5、微信小程序、Android、iOS 等十余种端。

## 七、项目特色与创新点

1. **以"宠物"为身份的社交模型**，不同于以人为中心的传统社交平台。
2. **社区 + 商家 + 活动 + 积分** 四位一体的业务闭环设计。
3. **多端 + 多库 + 多服务** 的现代 Web 架构实践，体现微服务与前后端解耦理念。
4. **用户体验细节**：踩 / 不感兴趣 / 拉黑等多维过滤机制，避免单一推荐导致的内容疲劳。
5. **可扩展性**：每个模块独立 `package.json` 与端口，可单独部署伸缩。

## 八、技术总结

本项目完整覆盖了一个生产级 Web 应用的关键技术栈：Node.js / Express / Sequelize / MySQL / MongoDB / JWT / Vue 3 / Vite / UniApp / Element Plus。从需求分析、数据库设计、后端 API、跨端前端、后台管理，到商家业务、积分激励与微服务拆分，均进行了完整的工程实践，为后续毕业课题及实际工程开发奠定了扎实基础。
