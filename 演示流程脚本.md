# 宠物社交平台（PetSocial）系统演示流程脚本

> 建议演示总时长：5~7 分钟。演示前确保 4 个模块已全部启动且数据库可访问。

---

## 一、演示前准备

### 1. 启动顺序（终端 4 个窗口）

```bash
# 1) MySQL / MongoDB 服务已启动（本地或远程）

# 2) 主后端
cd server && npm start          # 监听 3001

# 3) 百科微服务
cd dataDB && npm start          # 监听 3005

# 4) 移动端（H5 演示用）
cd frontend && npm run dev:h5   # 浏览器自动打开

# 5) 后台管理
cd admin && npm run dev         # http://localhost:5173
```

### 2. 准备账号

| 角色 | 账号 | 密码 / 验证码 | 说明 |
|------|------|---------------|------|
| 普通用户 A | `userA@xxx.com` | 邮箱验证码 | 演示发帖、点赞、积分等 |
| 普通用户 B | `userB@xxx.com` | 邮箱验证码 | 演示关注、互动 |
| 商家 | `merchant@xxx.com` | 邮箱验证码 | 演示商家入驻、礼品上架 |
| 管理员 | `admin@petsocial.com` | 邮箱验证码 | 演示后台审核 |

> 提前在数据库中预置数据，避免现场注册等待邮件。

### 3. 准备演示素材

- 1~2 张宠物图片用于发帖。
- 1 个商家头像与 1 张礼品图。
- PPT 切到"系统演示"页之前，把所有窗口排好（建议双屏：左屏移动端 H5，右屏后台 + 终端）。

---

## 二、演示主流程（按场景）

### 场景 1：用户注册与登录（约 30s）

1. 打开移动端 H5，进入"登录"页。
2. 切换到"注册"，输入邮箱 → 点击"获取验证码"（演示调用 `/api/account/send-code`）。
3. 输入验证码完成注册，自动跳转登录页。
4. 用 A 账号登录，**展示控制台**：JWT Token 已写入 storage。

> **答辩亮点话术**："这里使用的是邮箱验证码 + JWT 的无状态鉴权方案，Token 默认有效期 7 天，所有受保护接口在 `auth.js` 中间件中统一拦截。"

---

### 场景 2：发布与浏览帖子（约 60s）

1. 进入"发布"页，输入标题、内容，上传 1 张宠物图。
2. 选择话题"萌宠日常"和分类"日常生活"。
3. 提交后回到首页，展示帖子已出现在信息流顶部。
4. 进入帖子详情，演示：
   - 点赞（PostLike）
   - 评论（嵌套回复）
   - 收藏（Bookmark）
   - 长按帖子 → "不感兴趣" 过滤

> **答辩亮点话术**："我设计了多维度反馈机制，除了点赞，还有踩、不感兴趣、拉黑作者，这些都会影响后续的信息流过滤。"

---

### 场景 3：关注与社交（约 30s）

1. 切换 B 账号登录，进入用户 A 的主页，点击"关注"。
2. 回到 A 账号，进入"我的关注"和"粉丝"列表，展示 B 已出现。
3. 演示拉黑：A 拉黑 B，B 主页对 A 不可见。

> **答辩亮点话术**："Follow 表是 Pet 表的自连接多对多关系，BlockPet 也类似，做到了双向不可见。"

---

### 场景 4：积分系统与商城兑换（约 60s）

1. 用 A 账号查看"积分中心"，展示积分流水（PointLog）。
2. 进入"积分商城"列表，挑选一个礼品 → "立即兑换"。
3. **重点展示**：兑换成功后积分实时减少（修复后的 commit 8f0e806）。
4. 进入"我的兑换"列表，查看兑换记录。

> **答辩亮点话术**："积分变动全部写入 PointLog 流水可追溯，兑换接口是事务操作，保证扣分和生成兑换记录的原子性。"

---

### 场景 5：商家入驻与礼品管理（约 45s）

1. 切换商家账号，进入"申请商家"页面 → 提交申请。
2. 切换到 **管理员后台**（`http://localhost:5173`），打开"商家审核"，通过申请。
3. 切换回商家端，进入"礼品管理" → "新增礼品"，填写所需积分、库存、图片。
4. 上架成功后回到 A 账号商城，展示新礼品。

> **答辩亮点话术**："商家、礼品、兑换是一条完整的链路，全部在 Sequelize 关联中定义，权限隔离通过 role 字段判断。"

---

### 场景 6：线下活动报名（约 30s）

1. 商家发布一场线下活动（活动表单页）。
2. 用 A 账号进入"活动列表"，查看刚发布的活动。
3. 进入活动详情 → "立即报名"，展示 ActivityParticipant 写入。

---

### 场景 7：宠物百科微服务（约 20s）

1. 移动端进入"百科"页，浏览 dataDB 服务返回的条目。
2. **简短说明**："这是一个独立的微服务，使用 MongoDB 存储非结构化百科文档，端口 3005。"

---

### 场景 8：后台管理系统（约 60s）

切换到管理员后台，依次演示：

1. **帖子管理**：搜索 / 删除违规帖子。
2. **评论管理**：删除评论。
3. **话题 / 分类配置**：新增、编辑、排序。
4. **商家审核**：通过 / 拒绝商家申请。
5. **活动管理**：管理线下活动。

> **答辩亮点话术**："后台系统使用 Element Plus 组件库构建，通过 Vite 代理到 3001 端口，开发期无需 CORS 配置。"

---

## 三、收尾（约 10s）

- 切回 PPT，进入"创新点与总结"页。
- 总结陈词："以上就是 PetSocial 的核心功能演示，整个系统覆盖了从用户社交、内容生产、积分激励到商家服务和线下活动的完整闭环。"

---

## 四、应急预案

| 场景 | 处理方式 |
|------|----------|
| 邮件验证码收不到 | 提前用预置账号，绕过验证码环节 |
| 接口 500 报错 | 切到终端日志快速定位，必要时跳过该步骤 |
| 网络异常 | 启用 `localhost` 的本地数据，避免外网依赖 |
| 微信小程序加载慢 | 演示用 H5，备用窗口提前打开 |
| 后台跨域报错 | 确认 admin 的 `vite.config.js` 代理配置正确 |

---

## 五、演示节奏建议

- **快进**：注册 / 上传 / 表单填写等纯 UI 操作可加速。
- **强调**：核心业务点（积分扣减、关联关系、流水可追溯）应明确停顿讲解。
- **联动**：演示时同步展示**右侧终端**的 SQL 日志或后端打印，体现"看得见的数据库交互"。
- **节奏**：每个场景 30~60 秒，确保 5~7 分钟内完成。
