# 宠物百科测试数据（3 条）

> 数据结构对应 `dataDB/models/Pet.js`，存入 MongoDB 集合 `pets`。
> 评分字段（sticky / shout / friendly / lint / beauty / odour / saliva / active）取值范围 1~5，数字越高代表该特性越强。

---

## 字段说明

| 字段 | 含义 |
|------|------|
| `name` | 中文名（必填） |
| `engname` | 英文名 |
| `nation` | 原产地 |
| `life` | 平均寿命 |
| `price` | 市场参考价格（元） |
| `pic` | 图片 URL |
| `sticky` | 粘人程度（1~5） |
| `shout` | 吠叫 / 鸣叫程度（1~5） |
| `friendly` | 对人友好度（1~5） |
| `lint` | 掉毛程度（1~5） |
| `beauty` | 美容护理需求（1~5） |
| `odour` | 体味（1~5） |
| `saliva` | 口水量（1~5） |
| `active` | 活跃度（1~5） |
| `message` | 性格特点 |
| `feed` | 喂养建议 |
| `relative` | 近亲 / 相似品种 |
| `pick` | 挑选要点 |

## 一键导入示例

### 方式一：通过 API 接口（推荐）

服务启动后（`cd dataDB && npm start`），执行以下任一命令逐条插入：

```bash
# Windows PowerShell
curl -Method POST -Uri http://localhost:3005/api/pets `
     -ContentType "application/json" `
     -Body (Get-Content -Raw .\pet1.json)

# Bash / cmd
curl -X POST http://localhost:3005/api/pets ^
     -H "Content-Type: application/json" ^
     -d @pet1.json
```
