## 一、选题背景与意义（P3）

- 宠物经济持续升温，2025 年中国宠物市场规模超 3000 亿元（可补充数据来源）。
- 现有社交平台缺少宠物垂直场景的深度功能。
- 痛点：分享渠道分散、商家与用户脱节、缺乏积分激励与线下活动闭环。
- 选题价值：垂直社交 + 商家服务 + 积分激励 + 线下活动一体化。

---

## 二、需求分析（P4）

- **用户角色**：普通用户（铲屎官）、商家、管理员。
- **核心用例**：注册登录、发布帖子、关注互动、积分兑换、活动报名、内容审核。
- **非功能需求**：跨平台（H5 / 小程序 / App）、可扩展、安全（JWT 鉴权）、易维护。

> 可放一张用例图 / 需求脑图。

---

## 三、系统架构（P5）

- **多端分离 + 微服务** 整体架构图（建议手绘或使用 draw.io）。
- 四大模块：
  - `server/`：Express 5 + MySQL + Sequelize（端口 3001）
  - `frontend/`：UniApp + Vue 3（H5 / 微信小程序 / App）
  - `admin/`：Vue 3 + Vite + Element Plus（端口 5173）
  - `dataDB/`：Express + MongoDB（端口 3005）

> 关键句："关系型 + 文档型 双数据库，合适的数据用合适的库"。

---

## 四、技术栈选型（P6）

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 后端 | Express 5 + Sequelize | 生态成熟、ORM 关联清晰 |
| 数据库 | MySQL + MongoDB | 关系数据 + 非结构化百科 |
| 鉴权 | JWT + Nodemailer | 无状态鉴权 + 邮箱验证 |
| 前端 | Vue 3 + UniApp | 一套代码多端运行 |
| 后台 | Element Plus + Vite | 组件库丰富、构建快 |

## 六、后端 API 设计（P9）

- 路由按领域划分（13 个 API 路由文件）：
  - `/api/account`、`/api/pet`、`/api/post`、`/api/comment`
  - `/api/follow`、`/api/topic`、`/api/category`、`/api/bookmark`
  - `/api/point`、`/api/upload`
  - `/api/merchant`、`/api/gift`、`/api/activity`
- **统一约定**：JSON 响应格式 `{ code, msg, data }`，`code=0` 为成功，`code=401` 触发前端登出。
- **鉴权**：`server/src/middleware/auth.js` 集中拦截。

## 十、项目特色与创新点（P16）

1. 以"宠物"为身份的社交模型。
2. 社区 + 商家 + 活动 + 积分 四位一体业务闭环。
3. 多端 + 多库 + 多服务 现代 Web 架构。
4. 多维内容过滤（踩 / 不感兴趣 / 拉黑）提升体验。
5. 模块独立部署，便于扩展。

---

## 十一、测试与部署（P17）

- 测试方式：Postman 接口测试、浏览器多端预览、微信开发者工具调试。
- 部署方案：Node 进程 + Nginx 反向代理；MySQL / MongoDB 独立实例；前端静态资源 CDN。
- 数据自举：启动时 `sequelize.sync({ alter: true })` 自动建表并初始化数据。

---

## 十二、不足与展望（P18）

- 当前不足：暂无单元测试覆盖、推荐算法较简单、未实现 IM 即时通讯。
